GitHub Pages Setup Guide

Follow these steps to deploy your website to GitHub Pages for free

1Create a GitHub Account

If you don't have one already, go to github.com and sign up for a free account.

2Create a New Repository

Click on the "+" icon in the top right corner and select "New repository".

Name it your-username.github.io (replace "your-username" with your actual GitHub username).

Make sure to set the repository to Public and initialize with a README.

3Upload Your Website Files

Click on "Add file" ‚Üí "Upload files" and upload all your HTML, CSS, and JavaScript files.

Your main page should be named index.html for it to work properly.

4Enable GitHub Pages

Go to your repository's Settings tab.

In the left sidebar, click on Pages.

Under "Source", select Deploy from a branch and choose the main branch.

Click Save.

5Wait for Deployment

It may take a few minutes for your site to be published. Refresh the page until you see the green success message with your website URL.

Your website will be available at https://your-username.github.io

6Custom Domain (Optional)

If you have a custom domain, you can add it in the GitHub Pages settings under "Custom domain".

You'll also need to configure your DNS settings with your domain registrar.

Note: GitHub Pages is completely free and perfect for static websites like yours. It automatically handles SSL certificates for secure HTTPS connections.
